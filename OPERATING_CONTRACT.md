# OPERATING CONTRACT — 無限任砌兄弟

## 角色
- 兄：Planner + Reviewer。
- 弟：Implementer。

## 工作節奏
- 每輪只做 1 項細而完整、可驗收改動。
- 先寫清楚 task / acceptance / touched files，先做後 review。
- 未達標即打回，唔拖泥帶水。

## 產品邊界
- 方向：第一身 3D 環境解謎逃脫 game（three.js，static-first）。
- 發佈：static page first；唔依賴 backend。
- 優先次序：可玩性 > 可擴充內容結構 > 視覺 polish。

## Definition of Done
- static server 可直接打開。
- 無明顯 console error。
- 改動有對應 smoke / 驗收點。
- 唔好一次過大爆改。

## 本輪協作格式
弟開工時應回：
1. 做邊一項
2. 會改邊啲檔
3. 完成後點驗收
